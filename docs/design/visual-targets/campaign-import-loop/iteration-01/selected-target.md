# Selected Target

Selected direction: combine Concept C's compact mobile-first import console with Concept B's stronger bounded diagnostics bay.

## Binding

- Bottom command shelf remains the shell route navigation model.
- The top shell stays reserved for Directive identity/status/global actions.
- Package Library and Latest Import are page-local management groups inside Campaign.
- The dropzone and Choose File browse control are the only import action in this target and must read as real controls.
- Library readiness appears as compact status blocks before the import action.
- Latest import diagnostics are bounded and visually subordinate to the import controls unless there is an issue.
- Issues use muted warning/danger treatment and a capped list.
- Package IDs and issue messages wrap in structured text lines instead of clipping.
- Desktop may use a two-column library/diagnostics layout when width allows.
- Phone stacks status, import controls, and latest result in stable rows with small modern typography.

## Not Binding

- Generated concept text.
- Exact colors or fictional package names.
- Decorative rails that look like disabled controls.
- Any concept layout that turns Library or Import into shell route tabs.
- Any extra hidden diagnostics or raw JSON invented by the image model.

## Runtime Acceptance Notes

- Preserve Choose File label, drag/drop behavior, and hidden file input behavior.
- Preserve existing package/start/resume/load action semantics.
- Desktop and phone screenshots must show zero top route tabs.
- Live layout metrics must report no clipped labels.
