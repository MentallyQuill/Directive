# Selected Target

Status: selected for implementation.

Selected direction: combine Concept C's mobile-first roster density with Concept B's command-chain emphasis. Bottom route navigation is binding; top route navigation is outside the target space for this runtime shell.

Binding traits:

- Bottom navigation remains the primary route model.
- The top shell remains limited to Directive identity/status and global Back/Close actions.
- Crew begins with compact readiness/status blocks, not a tall summary card.
- Officer records become dense LCARS roster rows, not generic metadata cards.
- Officer name and billet are the first scan targets.
- Rank/species/continuity are compact chips or rows.
- Long role summaries wrap safely without dominating the row.
- Continuity is player-safe: tracked/initialized/player character, never hidden values.
- Font scale stays smaller and modern with stable line heights.

Non-binding traits:

- Exact generated text.
- Exact color values.
- Fictional headshots or portraits.
- Decorative micro-labels that do not map to runtime state.
- Any control layout that would break SillyTavern or Lumiverse host constraints.
