# Mission Recovery Concept Prompts

Shared base prompt:

Create original UX concept art for Directive's Mission Recovery section inside a compact SillyTavern extension shell. The target helps a player protect and repair campaign state with Save Controls, Save Game, Save As, Narration Recovery, Last Outcome review, Rewrite Narration, Rerun Outcome, and Delete Outcome. Use an original LCARS-inspired visual identity with asymmetric rounded rails, black glass surfaces, amber/coral/plum/teal/blue accents, compact status blocks, and crisp small modern typography. Do not use official logos, screenshots, copied Star Trek graphics, exact episode displays, unreadable filler text, hidden campaign facts, or decorative fake controls.

Route navigation belongs in the persistent bottom command shelf. There is no requirement for primary menu navigation to be on the top. The top shell is only for Directive title/status and global actions. Mission-local Command / Context / Side Work / Recovery sub-tabs may appear inside the Mission content when they improve scanability.

Do not sacrifice UX for LCARS aesthetics. Recovery controls must be easy to trust under pressure. Safe save actions, narration repair actions, reversible outcome actions, and destructive Delete Outcome must not blur together. No clipped labels, no overlapping bottom navigation, no giant hero text, and no fake controls.

## Concept A

Conservative Saga-close LCARS. A compact right-side desktop extension panel with Recovery selected, four small status tiles for Save, Branch, Narration, and Outcome, a Save As row, Save Game / Save As buttons, and a quiet area for Narration Recovery or Last Outcome when available. Keep density close to Saga mobile while translating the section into LCARS panels.

## Concept B

Rich command-console LCARS. A stronger recovery dashboard with asymmetric rails and grouped operational panes: Save State, Branch Snapshot, Narration Repair, Last Outcome Review, and Risk Actions. Separate safe, repair, rerun, and destructive actions through structural color-blocking and compact labels.

## Concept C

Compact mobile-first LCARS and preferred target. A 390px phone shell with Recovery selected, two-column status tiles for Save, Branch, Narration, and Outcome, a concise Save As input, grouped safe actions, and a clearly separated Risk Actions row for Delete Outcome. The persistent six-tab bottom command shelf is the route menu. This concept should bias the code pass.
