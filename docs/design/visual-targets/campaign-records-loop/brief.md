# Campaign Records Visual Target Loop

Target unit: Campaign page record management.

Target scope: Creator Drafts, Saves, current-save highlighting, package library/import controls, and record action rows.

Navigation direction: bottom route navigation remains the only shell route model. Records, Library, and Import are page-local management groups, not shell routes.

LCARS direction: compact original LCARS-inspired records console with status tiles, dense rows, clear current-save state, and touch-safe Load/Resume/Import actions.

UX direction: make the record list scannable before making it decorative. Long save names and draft package names should wrap cleanly or be structured into title/meta lines. The user should see record counts and the most relevant records without a tall generic stack.

Acceptance:

- Before/after screenshots cover desktop and phone-width record management.
- The records surface exposes compact status tiles for drafts/saves/library state.
- Creator Drafts and Saves rows use LCARS record styling instead of generic metadata rows.
- Current save state is visibly distinct.
- Load and Resume actions remain real buttons with unchanged labels.
- Bottom route navigation remains six tabs with zero top route tabs.
- Phone screenshots show no clipped labels, odd wrapping, or overlap with the bottom shelf.
