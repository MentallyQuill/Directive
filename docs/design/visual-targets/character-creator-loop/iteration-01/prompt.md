# Iteration 01 Prompts

Shared base:

Create original UX concept art for Directive's Character Creator page inside a compact SillyTavern extension shell. The page creates a player officer for a starship campaign. Use an original LCARS-inspired visual identity with asymmetric rounded rails, black glass surfaces, amber/coral/plum/teal accents, and crisp small modern typography. Do not use official logos, screenshots, copied Star Trek graphics, exact episode displays, unreadable filler text, or decorative panels that look like unusable controls.

Navigation rule:

Route navigation belongs in the persistent bottom command shelf. Do not put route navigation in the top shell. The top shell is only for Directive title/status and global actions. Back may appear as a narrow integrated bottom shelf segment when route history exists. Page-local creator steps may sit inside the content.

UX rule:

Do not sacrifice UX for LCARS aesthetics. Controls must be readable, touch-safe, and visually distinct from decoration. Avoid clipped labels, overlapping text, giant hero typography, hidden invented state, and fake controls.

## Concept A

Conservative Saga-close LCARS personnel commissioning console. Show a compact right-side desktop extension panel with a commissioning file header, four status tiles for Identity, Service, Personality, and Review, a selected Identity step with form fields, and a command row for Save Draft, Begin, Back, and simulation mode. Keep layout close to the existing Saga mobile density while translating it through LCARS paneling.

## Concept B

Rich command-console LCARS personnel commissioning console. Use stronger LCARS geometry, denser asymmetric rails, and a more dramatic status hierarchy while preserving real form usability. Show package/role/draft status, progress tiles, active form section, review readiness, and a bottom route shelf. Make the command area feel like a starship operations panel without turning form fields into decoration.

## Concept C

Compact mobile-first LCARS personnel setup screen in a 390px phone-width shell. The first viewport shows title/status, four compact progress chips, a horizontal step selector, the active Identity form section, and a sticky command row with Save Draft, Begin, Back, and simulation mode. The persistent route navigation is one bottom command shelf with six tabs; Back is a narrow integrated segment only when available. This is the preferred direction for this pass.
