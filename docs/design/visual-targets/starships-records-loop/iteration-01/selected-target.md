# Selected Target

Selected direction: Concept C, compact mobile-first Starships records console, with Concept A's conservative clarity.

Binding:

- Bottom route navigation stays the shell route model.
- Record management is a page-local Starships surface.
- Status tiles summarize Drafts, Saves, Current, and Library/Import state.
- Creator Drafts and Saves use compact LCARS record rows.
- Current save state is visually distinct.
- Load, Resume, and Import Package remain real buttons.
- Long record names use title/meta hierarchy and wrapping rather than clipped pills.
- Phone layout must avoid text clipping and bottom shelf overlap.

Not binding:

- Exact generated text.
- Exact concept-art colors.
- Decorative labels from image concepts.
- Fake segmented controls if they do not map to real behavior.
