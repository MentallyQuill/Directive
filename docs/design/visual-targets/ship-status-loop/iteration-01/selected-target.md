# Selected Target

Status: selected for implementation.

Selected direction: combine Concept C's mobile-first ship-status density with Concept B's operations/engineering readiness hierarchy. Bottom route navigation is binding; top route navigation is outside the target space for this runtime shell.

Binding traits:

- Bottom navigation remains the primary route model.
- The top shell remains limited to Directive identity/status and global Back/Close actions.
- Ship begins with a compact identity band and readiness blocks.
- Raw command IDs are replaced with player-facing command labels when available.
- Condition prose is subordinate to status blocks and wraps safely.
- Technical debt is grouped as operational caveats below the immediate readiness state.
- Font scale stays smaller and modern with stable line heights.

Non-binding traits:

- Exact generated text.
- Exact color values.
- Fictional ship schematics or portraits.
- Decorative micro-labels that do not map to runtime state.
- Any control layout that would break SillyTavern or Lumiverse host constraints.
