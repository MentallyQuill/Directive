# Selected Target

Status: selected for implementation.

Selected direction: combine Concept C's mobile-first latest-entry review with Concept B's command-history timeline rail. Bottom route navigation is binding; top route navigation is outside the target space for this runtime shell.

Binding traits:

- Bottom navigation remains the primary route model.
- The top shell remains limited to Directive identity/status and global Back/Close actions.
- Log begins with compact status blocks.
- Latest summaries appear before committed-input audit detail.
- Assisted-summary JSON is parsed into player-facing title, summary, and highlights.
- Long source IDs are de-emphasized as recorded/pending source status.
- Visible consequences and committed inputs remain available but compact.
- Font scale stays smaller and modern with stable line heights.

Non-binding traits:

- Exact generated text.
- Exact color values.
- Fictional stardates or event titles.
- Decorative micro-labels that do not map to runtime state.
- Any control layout that would break active SillyTavern constraints or obvious future-host adapter constraints.
