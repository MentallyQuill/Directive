# Mission Side Work Concept Prompts

Shared base prompt:

Create original UX concept art for Directive's Mission Side Work section inside a compact SillyTavern extension shell. The target helps a player handle optional starship command work: Open Orders, Follow-Up Opportunities, scheduled side missions, active assignment scenes, progress intervals, and the no-side-work state. Use an original LCARS-inspired visual identity with asymmetric rounded rails, black glass surfaces, amber/coral/plum/teal accents, compact status blocks, and crisp small modern typography. Do not use official logos, screenshots, copied Star Trek graphics, exact episode displays, unreadable filler text, hidden campaign facts, or decorative fake controls.

Route navigation belongs in the persistent bottom command shelf. There is no requirement for primary menu navigation to be on the top. The top shell is only for Directive title/status and global actions. Mission-local Command / Context / Side Work / Recovery sub-tabs may appear inside the Mission content when they improve scanability.

Do not sacrifice UX for LCARS aesthetics. Side Work must stay subordinate to the active mission while still making deferred tasks visible. Candidate cards must be readable, touch-safe, and easy to scan. Inputs and action buttons must remain visually distinct from decorative LCARS segments. No clipped labels, no overlapping bottom navigation, no giant hero text, and no fake controls.

## Concept A

Conservative Saga-close LCARS. A compact right-side desktop extension panel showing a Mission Side Work console with four small status tiles, a readable empty-state strip, and simple space for Open Orders or Follow-Up candidates when present. Keep density close to Saga mobile while translating the section into LCARS panels and segmented rows.

## Concept B

Rich command-console LCARS. A stronger operations-dashboard layout with an asymmetric LCARS rail, status tile cluster, Open Orders candidate stack, Follow-Up Opportunity stack, and progress interval strip. Make the section feel like a Starfleet operations queue without overpowering the main Mission command input.

## Concept C

Compact mobile-first LCARS and preferred target. A 390px phone shell with Side Work status tiles near the top of the Mission Side Work section, a deliberate empty-state console when no side work is active, and scalable compact cards for Open Orders, Follow-Ups, scheduled scenes, and progress intervals. The persistent six-tab bottom command shelf is the route menu. This concept should bias the code pass.
