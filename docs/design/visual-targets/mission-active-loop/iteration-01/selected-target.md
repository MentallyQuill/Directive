# Selected Target

Status: selected for implementation.

Selected direction: combine Concept C's mobile-first Mission loop with Concept B's stronger command-console hierarchy. Concept C's bottom-route model is binding; top route navigation is removed from the target space for this runtime shell.

Binding traits:

- Bottom navigation remains the primary route model.
- The top shell remains limited to Directive identity/status and global Back/Close actions.
- Mission page begins with compact status blocks, not a tall raw metadata card.
- Player Action or pending outcome review appears as the dominant command zone near the top.
- Save/Save As are available but subordinate to the mission command.
- Objectives, directives, pressure, and side-work sections are organized as operational context below the primary command.
- LCARS rails and segmented blocks group real workflow areas; decorative elements must not look interactive.
- Font scale stays smaller and modern with stable line heights.
- Long IDs/timestamps are de-emphasized or wrapped safely so they do not create cutoff, squished, or awkward text.

Non-binding traits:

- Exact generated text.
- Exact color values.
- Fictional status values invented by image concepts.
- Decorative micro-labels that do not map to runtime state.
- Any impossible control layout that would break active SillyTavern constraints or obvious future-host adapter constraints.
