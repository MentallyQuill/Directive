# Directive 0.3.0-pre-alpha.1

## Open-world and Scene Reconciliation revision

This pre-alpha release replaces the linear campaign runtime with schema-v2 open-world state and integrates Scene Reconciliation into the same deterministic transaction boundary.

Implemented:

- Persistent Asterion Reach world state, routes, factions, actors, fronts, clocks, and reaction rules.
- Concurrent authored and save-owned dynamic quest templates and instances.
- Grounded conversation-to-thread extraction with semantic deduplication, reinforcement, pacing, decay, expiry, closure, and promotion.
- Constrained Quest Architect and quest-action interpreter model roles; models propose structured data while deterministic code validates and commits it.
- Systemic objective-and-pressure resolution for quests without bespoke tactical graphs.
- Delegation, abandonment, expiry, transformation, and failure-forward quest lifecycle handling.
- Scene Reconciliation previews, immutable source anchors, causal invalidation, scratch-state replay, explicit acceptance/rejection, and prompt rebuilds.
- Salience-ranked, token-budgeted context planning for narrator-safe and Director-only audiences.
- Continuity Projection Matrix foundation: static continuity prompt lanes, source-backed Bronn/travel hard facts, sanitized projection diagnostics, advisory Director packets, host-native contradiction quarantine/recovery, focused factual-grounding canaries, and a five-user SillyTavern coordinator for bounded or full live certification.
- Schema-v2 campaign package and projection validators plus end-to-end dynamic-quest and reconciliation tests.

The bundled Ashes of Peace package uses the schema-v2 spine and the open-world Director Coordinator. No migration path from schema v1 is provided in this pre-alpha release.

Full 52-turn five-lane Continuity Projection Matrix live certification remains an opt-in soak run and is not implied by bounded `--turn-limit` evidence.
