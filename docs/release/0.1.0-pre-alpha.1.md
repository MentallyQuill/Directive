# Directive 0.1.0-pre-alpha.1 Release Notes

Directive `0.1.0-pre-alpha.1` is a pre-alpha development build. It exists to prove the first campaign loop, package contract, persistence boundary, and Mission Director architecture before public alpha polish.

Last updated: June 19, 2026.

## Compatibility

- Platform target: SillyTavern extension.
- Minimum SillyTavern version: `1.12.0`.
- Runtime namespace and extension key: `directive`.
- Automatic updates are disabled with `auto_update: false`.
- Storage and package formats may change in place during pre-alpha.

## Implemented Surface

- **Directive extension identity:** `manifest.json` points to `src/extension/index.js`, registers the `directive` key, and installs a **Directive** entry in the SillyTavern extensions menu.
- **Runtime shell:** the tabbed Directive window currently exposes **Campaign**, **Mission**, **Crew**, **Ship**, **Log**, and **Settings**; Campaign includes package library/import status and Settings includes basic state-safety controls.
- **Bundled starship package:** the Breckenridge/Ashes of Peace package is a schema-validated JSON package, not hardcoded runtime data.
- **Character Creator flow:** the Campaign tab can start or resume package-owned Character Creator drafts, save partial drafts, preserve the selected `Command` or `Exploration` simulation mode, accept the review, and create the first campaign save.
- **Campaign saves:** Save Game, in-panel Save As branch naming, Load Save, and stable-turn autosaves are implemented through the Directive storage repository.
- **Mission Director loop:** the Prelude mission, Chapter 1 through the Asterion/False Colors handoff, consequence packets, narrator packets, Command Log packets, and transaction-safe state deltas are covered by deterministic tests.
- **Command Competence:** Command Briefs, Domain Reports, Request Counsel, procedural warnings, authority notes, accepted risks, and no-gotcha checks are implemented across the current Chapter 1 MVP path.
- **Command Bearing:** Inspiration and Resolve tracks can support eligible intervention prompts, two-tier outcome improvement, anchored consequences, spend recording, and transaction rollback.
- **Pressure ledger:** Prelude and Chapter 1 outcomes can seed campaign-owned pressure records, route authored Open Orders candidates, persist Open Orders review choices, open selected assignments into active scene state, record first scene-beat progress, and resolve all three authored Open Orders I assignments with interval progress state without exposing hidden truth. The Long Repair and Borrowed Wings are the complete multi-beat MVP assignments.
- **Side opportunity detector:** committed post-Chapter-1 state can deterministically propose player-safe Missing Hardware Audit, Quarantine Review, and Pell Terms Follow-Up side-work candidates without provider calls, and Mission can schedule, defer, open, advance, resolve, or delegate them into campaign-owned follow-up state.
- **Side provider assistance contract:** fake structured provider output can propose player-facing follow-up phrasing and scene framing only after a deterministic candidate exists; Settings can run and inspect proposal-only diagnostics for eligible follow-ups; invalid JSON, provider failure, hidden leakage, and attempts to author state/source fields fail soft without mutating campaign state.
- **Package import workflow:** Campaign can import data-only `.directive-starship.zip` archives, normalize them into persisted package records, reject unsafe paths and active content, report diagnostics, and gate **Start Campaign** when imported runtime assets are incomplete.
- **Alpha gate:** `tools/scripts/run-alpha-gate.mjs` runs the dependency-free contract suite for the current runtime, package, storage, Director, competence, pressure, and repo-structure invariants.

## Pre-Alpha Boundaries

- Directive is not yet a public alpha release.
- Chapter 1 can hand off to Chapter 2: False Colors, Chapter 2 now has transparency-terms, Orison evidence-baseline, Aegis medical-trust, security-access demonstration, and joint investigation charter playable slices, Open Orders I can review/select authored candidates, open active assignment scenes, record scene-beat progress, and resolve all three authored first-interval assignments with satisfied/overextended interval state, and deterministic post-Chapter-1 follow-up side work can be scheduled, opened, advanced, resolved, or delegated. Provider-assisted follow-up proposal validation can route through runtime host generation, persist sanitized proposal/diagnostic records, and surface a Settings run/diagnostics control for eligible follow-ups; the live SillyTavern path has proven both fail-soft timeout diagnostics and accepted proposal-only persistence.
- Live SillyTavern confidence currently includes static source smoke, strict CSRF-bootstrapped `/api/files` upload/verify/read/delete smoke, repeatable browser route coverage through the Edge/Chrome CDP fallback, repeatable desktop/phone route screenshot geometry, repeatable Save Game / Save As branch reselect, and repeatable provider-backed Mission commit proof with `providerGeneration.proven === true`. Manual browser verification still covers creator mode persistence, post-Chapter-1 Follow-Up Opportunity scheduling and scene play, scheduled follow-up branch preservation, Settings provider-assist diagnostics/action surfacing with persisted timeout and accepted proposal diagnostics, Command Log review-row safety, mobile overlay behavior, and route inspection.
- Starship package import has a player-facing Campaign workflow for data-only archives, persisted package records, diagnostics, and runtime-asset gating; export, delete, update comparison, and richer trust review remain future work.
- Automatic SillyTavern message edit, delete, and branch interception remains future work; explicit runtime recovery controls exist.
- Broader host support is active through `src/hosts/`: SillyTavern and Lumiverse use separate adapters around the shared engine and shared command-spine shell, the Lumiverse descriptor targets the 1.0.4 Spindle extension surface with app-overlay mounting, and live Lumiverse model-call proof still depends on a valid local generation provider connection.
- Storage migrations are intentionally not maintained during pre-alpha. Update files and docs in place to the best current version.
