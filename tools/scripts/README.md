# Tool Scripts

Node-based local scripts for validation, smoke checks, audits, fixture tests, and release gates.

## Current Fast Checks

```powershell
node tools\scripts\run-alpha-gate.mjs
```

The alpha gate is the canonical maintained list. It is generated at runtime from [run-alpha-gate.mjs](run-alpha-gate.mjs), including bundled package, projection, crew dataset, and mission graph checks from the bundled package registry. Do not copy the full list here; update the gate script and focused notes below when coverage changes.

Coverage groups:

- extension shell, host contracts, SillyTavern adapters, lifecycle, events, storage, prompt, and generation routing
- runtime state, chat-native activation, turn orchestration, recovery, state transactions, sidecars, model-call authority, and prompt safety
- open-world package schema v2 contracts, quest/thread/story/reaction/director coordination, package import, and bundled package validation generated from the registry
- Command Bearing, Command Competence, mission graph validation, Director fixtures, transaction recovery, end conditions, and visual/system contracts

`smoke-sillytavern-live.mjs` is the local live-host scaffold for an active SillyTavern server. With no host configured, or with `--dry-run`, it prints the intended checklist and exits successfully. Set `SILLYTAVERN_BASE_URL=http://127.0.0.1:8000` to verify served Directive manifest/source assets. Set `DIRECTIVE_SILLYTAVERN_BROWSER=1` to check the live browser shell with Playwright when available or an installed Edge/Chrome CDP fallback; add `DIRECTIVE_SILLYTAVERN_TOGGLE_ONLY=1` to stop after proving the Directive enabled switch off/on path. Set `DIRECTIVE_SILLYTAVERN_SCREENSHOTS=1` with browser mode to capture desktop and phone-width PNGs for every Directive route and assert bottom-navigation geometry; `DIRECTIVE_SILLYTAVERN_SCREENSHOT_DIR` can override the output root. Set `DIRECTIVE_SILLYTAVERN_STORAGE=1` only when the host API is connected and the smoke may write, verify, read, and delete one smoke-owned `/user/files` JSON file; the script can bootstrap CSRF/session headers from `/csrf-token` when explicit env headers are absent. Browser-only no-generation UI smoke can run while SillyTavern reports API disconnected, but narration, provider routing, storage, preview/commit, and save/load require a connected API/provider surface.

`soak-sillytavern-campaign-live.mjs` is the preparation scaffold for the comprehensive live campaign soak. The current dry-run mode validates the Playwright-first harness assumptions, the unlimited model-call policy, the turn-settlement gate that treats `classifying` and `classified` as non-terminal states, the readable transcript policy, the roleplay-quality player input policy, the story-quality scoring policy, the factual-grounding policy for prompt availability, generated reply verdicts, material-fact audit levels, high-risk full-check triggers, and expected-facts-before-generation checklists, the generated player-safe fact canary packs and campaign-matrix live canary scripts for the active Ashes-first campaign matrix, the five-lane breadth-first parallel worker policy, the 5-10 turn Crew/Mission surface interval policy, the Command Bearing live certification schedule, interval playbook, minimum evidence threshold, closure proof levels, Mark Review gates, evidence/closure/Mark Review/spend policy, state-inspection requirements, failure severity policy, the artifact/report schema, the 52-turn phase contract, catastrophic and command-fitness End Conditions scenario contracts, the subtle command-conduct escalation ladders, strict warning-fail mode, release-certification summary output, bounded checkpoint artifacts, and optional served-extension freshness checks. Run `node tools\scripts\soak-sillytavern-campaign-live.mjs --dry-run`; add `--write-artifacts` to write the report skeleton under `artifacts/live-soak/sillytavern-campaign/<run-id>`, including `summary.md`, `report.json`, `snapshots/*.json`, `transcript/readable-chat.md`, `transcript/source-chat.jsonl`, `transcript/index.json`, `fact-checks/canary-index.json`, `campaign-matrix/canary-index.json`, `quality-review/scores.jsonl`, `quality-review/phase-summary.json`, `quality-review/manual-review-template.json`, `quality-review/manual-review-import.jsonl`, `quality-review/model-assisted-review/request.json`, `quality-review/model-assisted-review/result.json`, one player-safe fact canary pack per active campaign-matrix row, and one third-person campaign canary script per active campaign-matrix row. The JSON report and `summary.md` include `releaseCertificationSummary` with certification state, check/evidence counts, blockers, warnings, residual risk, and next action. Checkpoint artifacts under `snapshots/` store bounded counts, hashes, relative artifact paths, warnings/failures, and release-certification state without raw prompts, hidden state, secrets, or full transcripts. Use `--strict` or `DIRECTIVE_SOAK_STRICT=1` when warnings should make the final report fail. During live delegated smoke, the smoke runner captures per-turn prompt-inspection snapshots and immutable per-turn source transcript snapshots; the soak runner copies the latest SillyTavern-visible transcript into the top-level transcript artifacts, promotes delegated `turn-start`, `turn-end`, `transcript-capture`, and `prompt-inspection-capture` records into the top-level `live-log.jsonl`, runs deterministic story-quality scoring from the copied visible transcript into `quality-review/scores.jsonl` and `quality-review/phase-summary.json`, exports `quality-review/manual-review-template.json` for human readback scoring and can import completed manual entries into `quality-review/manual-review-import.jsonl` plus the main score ledger, prepares `quality-review/model-assisted-review/request.json`, invokes the real SillyTavern runtime `storyQualityReviewer` Utility role when live providers are available, imports accepted player-safe model scores into the same quality ledger, uses the paired prompt/transcript artifacts for per-generation factual-grounding checks under `fact-checks/<script-message-id>/fact-check.json`, also writes `fact-checks/transcript-level/fact-check.json`, invokes the real SillyTavern runtime `factualGroundingReviewer` role for model-assisted review, writes player-safe review request/output artifacts under `fact-checks/model-assisted-review/`, and records `quality-score`, `model-assisted-story-quality-review`, `fact-check`, plus `model-assisted-factual-review` live-log entries. The live runner should save the full player-visible chat for later story-quality review while keeping `live-log.jsonl` compact through previews, hashes, fact-check records, quality-score records, checkpoint records, and transcript pointers. In SillyTavern account mode, set `DIRECTIVE_SOAK_ST_USERS` so the preflight can authenticate as the first configured non-human soak user before reading protected extension files. `default-user` is reserved for human testing only and is rejected by automated soak preflights. Because extension fixes may land in parallel, live soak testing must begin by syncing the installed/served SillyTavern Directive extension copy; if the hash preflight is skipped, set `DIRECTIVE_CONFIRM_EXTENSION_SYNCED=1` only after syncing.

`run-continuity-matrix-five-user-soak.mjs` coordinates the Continuity Projection Matrix certification pass across the five Ashes soak lanes. Dry-run mode writes an aggregate plan under `artifacts/live-soak/continuity-projection-matrix-five-user/<run-id>`. Live mode first runs the five-user SillyTavern isolation preflight, then runs `soak-sillytavern-campaign-live.mjs` once per lane/user and aggregates only sanitized evidence: child artifact roots, check statuses, prompt keys, required CPM source ids, factual-grounding counts, stdout/stderr hashes, warnings, and failures. It fails if any live lane lacks the required CPM prompt keys, Bronn species/age source ids, the Breckenridge transit guard source id, or deterministic factual-grounding artifacts. Use `--turn-limit N` for bounded proof; omit it for the full 52-turn lane certification. Use `--resume` with the same `DIRECTIVE_CPM_FIVE_USER_SOAK_RUN_ID` after an interruption or late-lane failure; resumed runs reuse only lane artifacts whose requested turn depth matches and whose CPM prompt/source proof plus factual-grounding artifacts already pass. Example: `$env:SILLYTAVERN_BASE_URL='http://127.0.0.1:8000'; $env:DIRECTIVE_SOAK_ST_USERS='directive-soak-a,directive-soak-b,directive-soak-c,directive-soak-d,directive-soak-e'; $env:DIRECTIVE_LIVE_MODEL_CALL_BUDGET='unlimited'; node tools\scripts\run-continuity-matrix-five-user-soak.mjs --live --turn-limit 2 --write-artifacts`.

`run-command-bearing-closure-fixture-live.mjs` is the fixture-backed live certification runner for Command Bearing closure and Mark Review. It authenticates as a non-human SillyTavern soak user, compares the served extension against the checkout, opens a configured campaign save, creates a Save Game As branch, uses existing open Command Bearing evidence to create a durable quest closure candidate, runs the real SillyTavern Command Bearing evaluator model call, commits accepted review records into the isolated branch save, reloads the branch, and writes a report plus live-log entry. It requires `DIRECTIVE_SILLYTAVERN_USER`, `DIRECTIVE_SILLYTAVERN_RESUME_SAVE_ID`, and `DIRECTIVE_COMMAND_BEARING_FIXTURE_WRITE_SAVE=1`; `default-user` is rejected.

`run-command-bearing-point-lifecycle-live.mjs` is the fixture-backed live certification runner for Command Bearing point lifecycle. It resumes an isolated branch with accepted Command Bearing review state, creates a new Save Game As branch, recovers one point, verifies duplicate recovery and reserve-cap no-awards, readies/cancels/readies the point, commits a valid spend, verifies the two-band outcome improvement and controlled narration constraints, saves, exports, and writes a report plus live-log entry. It requires `DIRECTIVE_SILLYTAVERN_USER`, `DIRECTIVE_SILLYTAVERN_RESUME_SAVE_ID`, and `DIRECTIVE_COMMAND_BEARING_POINT_WRITE_SAVE=1`; `default-user` is rejected.

`smoke-sillytavern-terminal-endings-live.mjs` is the current live End Conditions smoke for an active SillyTavern server. It requires `SILLYTAVERN_BASE_URL` and `DIRECTIVE_SILLYTAVERN_GENERATION=1` or `DIRECTIVE_LIVE_GENERATION=1`. It creates fresh Ashes campaigns, forces a terminal Breckenridge failure, verifies terminal checkpoint posting, then resolves Save as branch, Replay from checkpoint, Push On, and Keep this ending through chat replies while checking the end-condition ledger, branch records, continuation frames, conclusion metadata, final band, and model-call growth. Set `DIRECTIVE_SILLYTAVERN_TERMINAL_TRIGGER=command-fitness-ladder` to run the subtler command-conduct trigger path instead: public captain fight, bridge impairment, officer assault, then unlawful command usurpation, with the first three probes required to stay non-terminal before the final command-removal checkpoint.

`check-playwright-soak-readiness.mjs` is the offline Playwright readiness probe for the soak harness. It does not contact SillyTavern. It launches Chromium, drives a role-locator click, switches the soak desktop and phone viewports, and writes a fixture trace plus screenshots. By default it writes to a temp directory; add `--write-artifacts` to write under `artifacts/live-soak/sillytavern-campaign/<run-id>`. The same browser-control probe is also part of the soak dry run unless `DIRECTIVE_SKIP_PLAYWRIGHT_BROWSER_CHECK=1` is set.

`check-sillytavern-multi-user-soak-readiness.mjs` is the live preflight for parallel soak workers. Dry-run mode validates the configured user list and Playwright availability without contacting the host. Live mode requires `SILLYTAVERN_BASE_URL`, five non-human users in `DIRECTIVE_SOAK_ST_USERS` for full breadth-first soak coverage, and `--live`; it opens one Playwright browser context per ST user, logs in when required, writes a unique Directive `/user/files` probe per user, verifies each user sees only its own probe, deletes the probes, and writes artifacts under the soak run folder. The default full-run handles are `directive-soak-a`, `directive-soak-b`, `directive-soak-c`, `directive-soak-d`, and `directive-soak-e`. `default-user` is reserved for human testing only and must not appear in the automated user list. Use this before running parallel workers or before trusting per-user extension patch lanes.

`discover-sillytavern-message-mutation-live.mjs` is a read-only Playwright discovery probe for the safest future edit/delete automation path. With no `--live` flag it prints a dry-run report. Add `--write-artifacts` to store `discovery/message-mutation-discovery.json` under the soak artifact tree.
